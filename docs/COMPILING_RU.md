# Руководство по сборке

В этом разделе описано, как правильно скомпилировать `HikkiSdk` и подготовить DEX-файлы для дальнейшей работы.

## Сборка и извлечение DEX

Для вашего удобства мы создали специальные Gradle-задачи, которые не только собирают проект, но и автоматически
извлекают нужные `.dex` файлы из готового APK.

### 1. Релизная сборка (Release)

Используется для финальной версии. В этом режиме применяется полная оптимизация R8, а все отладочные логи удаляются для
повышения производительности и скрытности.

```bash
./gradlew extractReleaseDex
```

**Где искать файлы:** `app/build/outputs/apk/release/dex/`

### 2. Бета-сборка (Beta)

Оптимальна для этапа разработки и тестирования. В ней сохранены логи, а конфигурация Proguard упрощена, чтобы вам было
легче отлаживать код.

```bash
./gradlew extractBetaDex
```

**Где искать файлы:** `app/build/outputs/apk/beta/dex/`

---

### Сборка без извлечения

Если вам нужен просто готовый APK-файл, используйте стандартные команды:

- `./gradlew assembleRelease` — для релизной версии.
- `./gradlew assembleBeta` — для бета-версии.
